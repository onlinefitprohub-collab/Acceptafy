# Handoff: Acceptafy — Email Mastery Suite Redesign

## Overview

A full UI/UX redesign of Acceptafy, a dark-themed React SPA for email marketers and deliverability professionals. The redesign addresses 10 documented pain points — sidebar cognitive load, weak feature hierarchy, passive dashboard, dense results view, minimal alerts, poor mobile — by:

1. Replacing the 12+ item nested sidebar with an **icon rail + contextual column + ⌘K command palette**
2. Rebuilding the Dashboard as a **contextual "mission control"** with action-prompt surfacing
3. Reworking the Email Grader into a **split-panel input with live signals** + **prioritized Top-3-fixes results view**
4. Replacing per-section rainbow gradients with **a single restrained accent (green) + semantic warn/critical colors**
5. Giving Deliverability a **single-page health dashboard** with drill-downs
6. Upgrading alerts to **severity icons + deep-link "Fix in [tool]" CTAs**
7. Designing a **375px mobile layout** with bottom tabs and pinned primary CTAs

Scope covered (per original brief sections A–H): Navigation, Dashboard, Grader (input + results), Tool overview pages, Deliverability hub, Alerts panel, Mobile, Visual refinements + component inventory.

## About the Design Files

The files in this bundle are **design references created in HTML** — a hi-fi slide-deck prototype showing the intended look and behavior. **They are not production code to copy directly.**

The task is to **recreate these designs in the Acceptafy codebase's existing environment**: React 18 + TypeScript, ShadCN UI, Tailwind CSS, Lucide icons, Radix UI primitives. Use established patterns and composition rules from the existing app. **Do not introduce new frameworks or component libraries.**

The HTML prototype uses vanilla CSS variables, static SVGs, and JetBrains Mono/Inter Tight — these are illustrative. The real implementation should use Tailwind tokens (extended where noted), the existing type stack if already chosen, and the existing Lucide icons instead of inline SVGs.

## Fidelity

**High-fidelity.** All colors, spacing, typography, hierarchy, copy, and interaction intent are final. Recreate pixel-perfectly using ShadCN primitives where possible. Where the prototype uses custom SVG (the grade ring on the Results page, small chart sparklines), those are custom components described below.

## Design Tokens

### Colors (dark theme)

Add these as CSS variables / Tailwind theme extensions. All values are `oklch()` where noted for wide-gamut fidelity — fall back to hex is provided.

```css
/* Surfaces */
--background:    #0B0C0E;
--surface:       #14161A;   /* card */
--surface-2:     #1A1D22;   /* hover/raised */
--border:        #22262D;
--border-strong: #2C313A;

/* Text */
--foreground:        #ECEEF1;
--muted-foreground:  #9CA3AE;
--text-mute:         #5E6773;  /* tertiary labels */

/* Accent (single) — electric green */
--accent:       oklch(78% 0.18 145);      /* ≈ #4fd97f */
--accent-ink:   oklch(30% 0.10 145);      /* text on accent fills */

/* Semantic */
--warn:         oklch(80% 0.14 75);       /* amber   ≈ #e0b450 */
--critical:     oklch(70% 0.22 25);       /* red     ≈ #e6663a */
--info:         oklch(78% 0.11 230);      /* blue    ≈ #7eb7e0 */

/* Grade ramp — A..F */
--grade-a: oklch(78% 0.18 145);
--grade-b: oklch(80% 0.15 110);
--grade-c: oklch(82% 0.14 80);
--grade-d: oklch(75% 0.17 45);
--grade-f: oklch(70% 0.22 25);
```

**Critical change from existing app:** remove the per-section gradients (purple/pink for brand, blue/cyan Create, violet/purple Optimize, amber/orange Analytics, green/teal Deliverability, rose/pink Connections, emerald History). Color is reserved for semantic meaning: success (accent), warn, critical, and the grade ramp. Sections get icon identity only.

### Spacing, radius, shadow

```
radius:   xs 6 / sm 10 / md 14 / lg 20 / xl 28   (match Tailwind's rounded-md ≈ 14)
spacing:  Tailwind default
shadow:   card = inset 0 1px 0 rgba(255,255,255,0.03)
          elevated = 0 50px 120px -40px rgba(0,0,0,0.6)
```

### Typography

```
Sans:  Inter Tight, with font-feature-settings "ss01", "cv11"; letter-spacing -0.005em
Mono:  JetBrains Mono, tabular-nums ("tnum" 1)
Serif: Instrument Serif (decorative only — slide numerals)
```

|---|---|---|---|
| display | 54–104px | 700 | Marketing/cover only |
| h1 | 44px | 700, -0.025em | Dashboard greeting |

## Screens / Views

### A. Navigation architecture

**Layout:** three columns on desktop.

- **Rail** — 64px wide, dark (`#0E1013`), fixed. 8 domain buttons + logo on top, alerts bell + avatar at bottom. Active state: icon tinted accent, 3px accent strip on left edge, surface-2 background.
- **Contextual column** — 220–240px, dark (`#0E1013`), left border-separator. Title + "⌘N" keyboard hint at top. Tool list with icon + label + optional kbd badge. Group headers (eyebrow style). Footer contains XP/level, streak, progress bar.
- **Main content area** — fills remaining width. Top bar has breadcrumb on left (`Home / Overview`) + ⌘K search field (min 260px) + tier pill on right.

**8 rail domains:** Home, Grade & Results, Create, Optimize, Deliverability, Analytics, Academy, Connections. Alerts and user avatar are separate bottom group.

**⌘K palette** — existing ShadCN `<Command>` (cmdk). Globally available; placeholder copy "Grade email, rewrite subject, check domain…". Must be reachable by keyboard anywhere, and is the primary wayfinder on mobile.

**Rationale:** Marketers work in domain-bursts (grade → rewrite → re-grade). A fixed rail + contextual column keeps the burst's tools one click apart without hiding the full app. The rail never scrolls; nested sub-items collapse to contextual-column groups; advanced tools live behind an Advanced collapse rather than adding rail items.

### B. Dashboard (mission control)

**Top bar:** breadcrumb + ⌘K search + "Pro" tier pill.

**Hero row (2 columns, 1fr / 380px):**

- **Left:** eyebrow (date), h1 greeting "Good morning, Jess.", h3 sub "Two things need attention before your 11am send." Then a full-width **action prompt card** with a red-tinted background gradient (`color-mix(in oklab, var(--critical) 6%, var(--surface))`), a 36px square critical-icon chip, title "acceptafy.com was found on 2 blacklists", body explaining Spamhaus ZEN and Barracuda, Dismiss + primary "Open Domain Health →" CTAs.
- **Right:** XP card with accent tint. Shows "L7 · Email Grandmaster" label, streak badge ("🔥 12"), XP progress bar (72%, "1,820 XP · 680 to L8"), two mini stats (Next unlock, Achievements 14/42).

**Row 2 (2 columns, 1fr / 1fr):**

- **"Your most-used tools"** card with 2×2 inner grid. Each tile: small mono eyebrow ("⌘G · 42 USES"), h3 title ("Grade email"), description ("Paste → A–F in ~4s"). **Important:** title below eyebrow, not beside — prevents wrap-overlap in narrow tiles.
- **"Recent grades"** card. Rows of 44px grade badge + title/meta + score + relative timestamp. Grade badge uses grade-ramp color as fill; letter is 800 weight.

**State management:**
- `user.level`, `user.xp`, `user.xpToNext`, `user.streak`
- `alerts.critical[]` — drives the action prompt card (show top 1)
- `usage.byTool` sorted DESC → top 4 → rendered as the tiles
- `history.recent` limited to 3

**Contextual logic:** if no critical alerts, action prompt becomes a next-step suggestion ("Connect your ESP to unlock Campaign Stats"). If nothing applies, hide the card entirely — don't fill with generic quick-actions.

### C.1 Email Grader — input

**Layout:** 1.3fr / 1fr split inside a rounded container.

**Right (live signals):**

- **Spam triggers card** — counter badge, list of detected phrases with type labels

### F. Alerts panel

**Pattern:** ShadCN `<Sheet>` sliding from right, 440px wide.

- Header: title "Alerts" + count breakdown ("2 critical · 1 warning · 3 info") + filter pills (All / Critical / Unread)
- **Each row** structure:
  - 32px severity icon chip (critical = red `!`, warning = amber `△`, info = blue `i`, success = green `✓`) — tinted background
  - Right side: title + relative timestamp, description (muted), optional mono detail block, **"Fix in [tool] →"** deep-link accent text OR expanded action row with primary + ghost buttons
- Critical alerts get a subtle red-tinted row background AND expanded state by default
- Footer: total count + "Mark all read"

**State management:** `alerts.severity`, `alerts.createdAt` (for relative time via date-fns), `alerts.deepLink` (route + params), `alerts.expanded` (local), `alerts.readAt`.

## Interactions & Behavior

### Navigation
- Rail button click → route change, contextual column swaps to that domain's tools
- `⌘K` anywhere → open Command palette
- Contextual item click → route change, active state persists
- Alerts bell → open `<Sheet>` from right

### Grader
- Typing in textarea → debounce 400ms → re-run local heuristic signals, update Live Signals card
- Hovering a highlighted spam trigger span → `<HoverCard>` with explanation
- "Grade email" → submit → loading skeleton in results card → navigate to `/grade/:id`
- Clicking a Top-3 fix → navigate to destination tool with email payload in route state

### Alerts
- Row click → expand inline (Collapsible)
- "Fix in [tool] →" → close sheet, navigate with alert ID in query
- "Snooze 24h" → optimistic dismiss
- Critical alerts are sticky (cannot be auto-dismissed)

### Animations
- Grade ring stroke reveal: 600ms ease-out on mount
- Card hover: `translateY(-1px)` + `border-color` to accent-line at 160ms
- Sheet slide: Radix default (duration-200)
- Alert row expand: max-height 200ms ease
- Live-signal updates: cross-fade score number 180ms

### Loading
- Skeletons use `bg-surface-2` with shimmer
- Grader result: full card skeleton with preserved layout

### Empty states (to build)
- No grades yet → "Grade your first email" illustration placeholder + CTA + 3 example subject lines to try
- No alerts → "You're in the clear" with last-checked timestamp
- No ESP connected → prompt card on dashboard with ESP logos row
