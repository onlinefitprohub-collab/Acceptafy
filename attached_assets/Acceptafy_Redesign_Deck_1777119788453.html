Scope

Constraints

React 18 + TypeScript, ShadCN/Radix primitives, Tailwind, Lucide icons. No new frameworks. Dark-first, desktop primary (1280+), mobile supported (375+).

Dark · Restrained palette 11 slides
Design rationale

One accent, many jobs.

The current multi-gradient-per-section approach (blue Create, violet Optimize, amber Analytics…) trains the eye to treat color as decoration. That breaks when we need color to mean something — a critical alert, a failing score, a passed DNS check.

We reserve one accent (green) for success and CTAs, amber for warnings, red for critical. Sections get a silent icon identity only. Grades use a dedicated A–F ramp.

Principle 01
Progressive disclosure
Overall grade first. The top 3 fixes. Then everything else, collapsed.
Principle 02
Every screen has a next step
No "dead ends." Every result links to the tool that fixes it.
Principle 03
Grade in 2 clicks
⌘K from anywhere. Grader reachable in a single tap on mobile.
Principle 04
Earn the celebration
Gamification is ambient. Celebrate real milestones, not every action.

Nav rationale

Marketers work in domain-bursts: grade → rewrite → re-grade, or health → DNS → re-check. A fixed rail + contextual column keeps the burst's tools one click apart without hiding the full app. ⌘K is the power-user exit hatch and the primary wayfinder on mobile.

Mobile rules
collapsesRail → bottom tabs · contextual column → Sheet · breadcrumb → back arrow
stacksDashboard hero · grade ring + fixes · health summary (4-wide → 2×2)
defersRelated rail · Academy card · advanced tools (accessible via ⌘K)
pinsGrade email button, Fix with AI, primary CTA — always footer
